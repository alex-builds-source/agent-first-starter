# Security Policy

## Scope
This repository is public. Treat all content as publicly readable.

## Rules
- Never commit tokens, API keys, passwords, private keys, or sensitive logs.
- Use `.env.example` for placeholders only.
- Use `gitleaks git --pre-commit` before pushing.

## Reporting
If you find a security issue, open a private report to the maintainer account (`alex-builds-source`) via GitHub security advisory workflow when enabled.
